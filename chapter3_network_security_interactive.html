3.1

Cloud Network Architecture & Security

Cloud network architecture defines how resources, users, apps, and data are interconnected. It spans on-premises connectivity, internal cloud networking, and placement of security controls like firewalls, SGs, and VPNs.

🖥️
IaaS
Virtual servers, storage & networking you configure like a data centre. Most control, most responsibility.
🛠️
PaaS
Ready-made platforms (runtime, DB, dev tools). Developers focus on code, not infrastructure.
🌐
SaaS
Complete apps delivered over the internet (Gmail, Office 365). Least control, least responsibility.
🔗
Network Connectivity
Routers, firewalls, load balancers, VPNs connecting users to cloud resources securely.

3.2

VPC & Network Segmentation

Key Idea: A VPC is your private data centre inside a public cloud. Network segmentation divides it into isolated sections to limit the spread of attacks and meet compliance requirements.
🛡️
Isolation
Each VPC is logically separated from other customers' VPCs, even in the same account.
🗺️
Custom Networking
Choose CIDR blocks, define subnets, route tables, and gateways to suit your topology.
🔐
Security Controls
Security groups, ACLs, and firewall rules give fine-grained control over traffic.
📈
Scalability
Resources scale up or down on demand following cloud elasticity models.
Diagram — 3-Tier Segmented Architecture (E-Commerce)
🏗️ Types of Network Segmentation ▾
  • Physical Segmentation — Separate physical hardware; common in traditional networks but rare in public cloud.
  • Virtual Segmentation — Subnets, VLANs, and Security Groups within shared physical infrastructure. Dominates in cloud.
  • Application Segmentation — Limits which apps or services can talk to each other via micro-segmentation and service-to-service policies.
⚠️ Common Challenges & Solutions ▾
  • Misconfiguration — Incorrect rules expose resources. Fix: Use IaC templates + peer reviews.
  • Complexity — Large number of VPCs & rules become hard to manage. Fix: Maintain diagrams and docs.
  • Performance — Poor design creates bottlenecks. Fix: Monitor throughput and scale as needed.
  • Maintenance — Rules & certificates need ongoing updates. Fix: Schedule regular audits.
✅ Best Practices ▾
  • Separate VPCs for dev, staging, and production.
  • Clear naming conventions for all networking components.
  • Apply least privilege in SGs and NACLs — deny everything by default.
  • Automate VPC creation with IaC (Terraform / CloudFormation).
  • Monitor with VPC Flow Logs and cloud monitoring tools.
3.3

Firewalls, IDS & IPS

📦
Packet-Filtering
Examines headers only. Fast and stateless but can't inspect content deeply.
🔄
Stateful Inspection
Tracks connection state. Allows packets only if they're part of a legitimate session.
🕵️
Proxy / App-Level
Acts as intermediary at app layer. Hides internal IPs, enables deep inspection.
🚀
NGFW
Combines traditional firewall + DPI + IPS + app awareness + threat intelligence.

3.5

DoS / DDoS Attack Prevention in Cloud

💣
DoS (Single Source)
One machine floods a service with traffic until it can't handle legitimate requests.
🕸️
DDoS (Botnet)
Thousands of compromised machines attack simultaneously — much harder to block.
📡
Volume-Based
Consume all available bandwidth with massive UDP/ICMP floods.
⚡
Protocol Attacks
Exploit weaknesses — e.g., SYN floods exhaust server connection state tables.
🎯
App-Layer Attacks
Target specific endpoints (login, search) with expensive, repeated HTTP requests.
Diagram — Multi-Layer DDoS Defence Architecture
⚠️ Cloud-Specific DDoS Challenges ▾
  • Shared infrastructure — one tenant's attack traffic may affect others on the same host.
  • Elastic scaling cost — auto-scaling absorbs traffic but may cause unexpectedly high bills.
  • Global attack surface — attackers launch from many regions and IPs simultaneously.
  • IoT botnets — growth of insecure IoT devices has massively enlarged botnet capacity.
  • Mixed strategies — modern attacks combine volumetric + protocol + application-layer techniques.
⚠️ Remember: DDoS targets availability — not confidentiality or integrity. In cloud environments, elastic scaling can absorb attacks but may lead to very high bills if triggered by malicious traffic.
